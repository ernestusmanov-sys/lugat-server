# Lügat Server — Инструкция по развёртыванию

## Бесплатный деплой на Railway (5 минут)

### Шаг 1: Регистрация

1. Зайдите на [railway.app](https://railway.app)
2. Войдите через GitHub (создайте аккаунт если нет)

### Шаг 2: Загрузка сервера

1. Создайте репозиторий на GitHub, загрузите папку `lugat_server/`:
   ```
   lugat_server/
   ├── app.py
   ├── requirements.txt
   ├── Procfile
   ├── railway.toml
   └── dictionary.db   ← скопируйте вашу базу данных сюда!
   ```
2. В Railway нажмите **New Project → Deploy from GitHub repo**
3. Выберите репозиторий — Railway сам определит Python-проект

### Шаг 3: Настройка переменных окружения

В Railway: **Variables → Add Variable**:

```
ADMIN_TOKEN = ваш_секретный_пароль_администратора
```

> ⚠️ Обязательно смените ADMIN_TOKEN! Это пароль для модерации.

### Шаг 4: Получить URL сервера

После деплоя Railway покажет URL вида:
```
https://lugat-server-production.up.railway.app
```

Этот URL нужно вписать в приложении: кнопка **«Обновить словарь»** → поле «Адрес сервера».

## Альтернатива: Render.com (тоже бесплатно)

1. Зайдите на [render.com](https://render.com)
2. **New → Web Service → Connect GitHub repo**
3. Build Command: `pip install -r requirements.txt`
4. Start Command: `python -c "from app import init_db; init_db()" && gunicorn app:app`
5. Environment Variables: `ADMIN_TOKEN=ваш_пароль`

## API-эндпоинты

### Публичные (доступны пользователям)

| Метод | URL | Описание |
|-------|-----|----------|
| GET | `/api/version` | Текущая версия словаря |
| GET | `/api/updates?client_version=N` | Скачать обновление |
| POST | `/api/submit` | Отправить новое слово |

### Административные (требуют заголовок `X-Admin-Token`)

| Метод | URL | Описание |
|-------|-----|----------|
| GET | `/api/pending` | Список слов на модерации |
| POST | `/api/approve/<id>` | Одобрить слово |
| POST | `/api/reject/<id>` | Отклонить слово |
| POST | `/api/publish` | Опубликовать новую версию вручную |
| GET | `/api/stats` | Статистика |

## Как обновить словарь на всех клиентах

После того как вы добавили/исправили слова через административную панель:

1. Скопируйте обновлённый `dictionary.db` в папку сервера на GitHub
2. Сделайте commit/push — Railway автоматически перезапустит
3. Или вызовите эндпоинт вручную:
   ```
   POST https://ваш-сервер.railway.app/api/publish
   Headers: X-Admin-Token: ваш_пароль
   Body: {"note": "Добавлено 100 новых слов"}
   ```
4. Пользователи получат обновление при следующем запуске программы

## Проверка работы сервера

Откройте в браузере: `https://ваш-сервер.railway.app/`

Должен отобразиться JSON:
```json
{"name": "Lügat API", "status": "running", "version": "1.0"}
```

## Настройка в приложении (для администратора)

В конфиг-файле `data/sync_config.json` добавьте:
```json
{
  "server_url": "https://ваш-сервер.railway.app",
  "admin_token": "ваш_секретный_пароль",
  "auto_check": true
}
```

Или через интерфейс: **Обновить словарь** → введите URL → **Сохранить**.

## Размер и ограничения Railway (бесплатный план)

- Память: 512 МБ (достаточно)
- Диск: 1 ГБ (база словаря ~23 МБ — в порядке)
- Трафик: 100 ГБ/месяц (при 1000 пользователях — ок)
- Засыпает после 30 мин неактивности (первый запрос медленнее ~3 сек)

Для постоянной работы без засыпания — обновите до платного плана ($5/мес).
